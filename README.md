**Montazhka PRO** — это экосистема для управления рекламным агентством и координации работы монтажных бригад. Система объединяет веб-интерфейс для администраторов и Telegram-бота для исполнителей в единое рабочее пространство с синхронизацией данных в реальном времени.

---

## 🚀 Основные возможности

### 💻 Админ-панель (Web)
* **Безопасность:** Вход по персональному ПИН-коду с сохранением сессии (не нужно вводить пароль при каждой перезагрузке).
* **Управление заказами:** Создание, редактирование и удаление объектов.
* **Фильтрация:** Разделение заказов на "Активные", "Общие", "Архив" и "Все".
* **Поиск:** Мгновенный поиск по названию объекта.
* **Управление командой:** Добавление новых сотрудников и администраторов прямо из интерфейса.
* **Рассылки:** Массовая отправка уведомлений всем монтажникам в Telegram.
* **Статистика:** Наглядные счетчики активных и выполненных заказов.
* **Realtime:** Обновление данных на лету без перезагрузки страницы (Supabase Realtime).

### 🤖 Telegram-бот (Installer)
* **Авторизация:** Привязка Telegram ID к профилю через ПИН-код.
* **Личный кабинет:** Просмотр персональной статистики и активных задач.
* **Лента заказов:** Получение уведомлений о новых задачах и просмотр доступных объектов.
* **Фотоотчеты:** Возможность загрузки фотографий выполненных работ.

## 🛠 Технологический стек

* **Frontend:** [Next.js](https://nextjs.org/), [Tailwind CSS](https://tailwindcss.com/), [Lucide Icons](https://lucide.dev/).
* **Backend & DB:** [Supabase](https://supabase.com/) (PostgreSQL, Realtime).
* **Bot:** [Python](https://www.python.org/) + [Aiogram 3.x](https://docs.aiogram.dev/).
* **Deployment:** [Vercel](https://vercel.com/) (Web), [Railway/VPS](https://railway.app/) (Bot).

## ⚙️ Настройка и установка

### 1. Переменные окружения (.env)
Создайте файл `.env` в корневых папках проекта и укажите следующие ключи:

**Для Web-части:**
```env
NEXT_PUBLIC_SUPABASE_URL=ваш_url_supabase
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_ключ_supabase
NEXT_PUBLIC_TELEGRAM_BOT_TOKEN=токен_вашего_бота
Для Telegram-бота:

Фрагмент кода
TELEGRAM_BOT_TOKEN=токен_вашего_бота
SUPABASE_URL=ваш_url_supabase
SUPABASE_KEY=ваш_ключ_supabase
2. Структура базы данных
Система ожидает наличие двух таблиц в схеме public:

orders: поля id, title, description, status, assigned_to, is_general, images (array).

profiles: поля id (UUID), full_name, pin_code, role (admin/installer), telegram_chat_id.

🔑 Логика доступа
Администратор создает профиль на сайте, выбирает роль и назначает ПИН-код.

Монтажник находит бота, вводит ПИН-код — система автоматически привязывает его Telegram ID к базе.

Для доступа к админ-панели роль пользователя в таблице profiles должна быть admin.
