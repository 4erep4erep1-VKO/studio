# MontazhkaPRO v 3.0

Профессиональная система управления рекламными монтажами, состоящая из веб-панели для администраторов и Telegram-бота для исполнителей.

## Ключевые особенности

-   **Архитектура "Админка + Бот"**:
    -   **Веб-панель (Администратор)**: Полноценный web-интерфейс для создания, просмотра, редактирования, назначения заказов и управления пользователями. Доступна только для роли `admin`.
    -   **Telegram-бот (Монтажник)**: Упрощенный интерфейс для монтажников. Получение новых заказов, изменение статусов, отправка отчетов, получение уведомлений — все через Telegram.

-   **Разделение ролей**:
    -   **Администратор**: Полный доступ к управлению через веб-панель.
    -   **Монтажник**: Взаимодействие с системой исключительно через Telegram-бота.

-   **Надежная обработка данных и UX в веб-панели**:
    -   Отображение скелетонов во время загрузки данных для предотвращения "пустого экрана".
    -   Четкие сообщения об ошибках (отсутствие интернета, ошибки базы данных).
    -   Автоматическая перезагрузка данных при восстановлении интернет-соединения.
    -   Валидация файлов (размер до 5MB, тип) на стороне клиента перед отправкой на сервер.

## Технологии

-   **Веб-панель (Frontend)**: Next.js 15 (App Router), React, TypeScript
-   **Веб-панель (UI)**: Tailwind CSS, ShadCN UI, Lucide Icons
-   **Бэкенд & База данных**: Supabase (PostgreSQL, Storage, Auth)
-   **Telegram-бот**: Python, aiogram

## Развертывание и запуск

### 1. Настройка Supabase

1.  Создайте проект в [Supabase](https://supabase.com/).
2.  Получите URL проекта и `anon public key` из раздела `Project Settings` -> `API`.
3.  Выполните SQL-скрипты для инициализации базы данных. **Настоятельно рекомендуется использовать `supabase-reset.sql` для полной очистки и пересоздания структуры таблиц при первой настройке или в случае проблем.**
    -   Перейдите в `SQL Editor`.
    -   Откройте файл `supabase-reset.sql` в вашем проекте.
    -   Скопируйте все содержимое и выполните запрос в SQL Editor.
4.  Включите Realtime для таблиц `profiles`, `orders`, `comments`, `telegram_users` в разделе `Database` -> `Replication`.
5.  **Настройте политики RLS** (Row Level Security) для таблиц `profiles`, `orders`, `comments`, `telegram_users` в разделе `Authentication` -> `Policies`. Убедитесь, что политики соответствуют разделению ролей (например, админы могут читать все профили, пользователи только свои).

### 2. Запуск Веб-панели (Next.js)

1.  **Переменные окружения**: Скопируйте `.env.example` в `.env.local` и заполните:
    ```bash
    NEXT_PUBLIC_SUPABASE_URL=ВАШ_SUPABASE_URL
    NEXT_PUBLIC_SUPABASE_ANON_KEY=ВАШ_SUPABASE_ANON_KEY
    SUPABASE_SERVICE_ROLE_KEY=ВАШ_SUPABASE_SERVICE_ROLE_KEY # Используется на сервере для админских операций
    ```
2.  **Установка зависимостей**:
    ```bash
    npm install
    ```
3.  **Запуск в режиме разработки**:
    ```bash
    npm run dev
    ```
4.  **Развертывание на Vercel (Production)**:
    -   Подключите ваш репозиторий на GitHub к [Vercel](https://vercel.com/).
    -   Vercel автоматически будет деплоить при каждом `push`.
    -   Добавьте переменные окружения `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` в настройках проекта Vercel.

### 3. Запуск Telegram-бота (Python)

1.  **Создайте бота в BotFather**: Получите `TELEGRAM_TOKEN`.
2.  **Переменные окружения в `telegram_bot/bot.py`**:
    -   Откройте файл `telegram_bot/bot.py`.
    -   Замените `TELEGRAM_TOKEN` на ваш токен от BotFather.
    -   Замените `SUPABASE_URL` и `SUPABASE_KEY` на соответствующие значения (для бота пока можно использовать `ANON_KEY`, но в дальнейшем рекомендуется использовать `SERVICE_ROLE_KEY` для более безопасных серверных операций).
3.  **Перейдите в директорию бота**:
    ```bash
    cd telegram_bot
    ```
4.  **Установка зависимостей Python**:
    ```bash
    python3 -m pip install aiogram supabase
    ```
    (Если `pip` не найден, убедитесь, что в `.idx/dev.nix` добавлен `pkgs.python311Packages.pip` и окружение пересобрано.)
5.  **Запуск бота**:
    ```bash
    python3 bot.py
    ```
    Бот будет запущен и готов к работе.

# MontazhkaPRO v 3.0

Профессиональная система управления рекламными монтажами.

## Ключевые особенности

- **Разделение ролей**:
  - **Администратор**: Полный доступ к созданию, редактированию и управлению заказами и пользователями.
  - **Монтажник**: Просмотр назначенных заказов, обновление статусов и загрузка отчетных файлов.

- **Единый пул заказов с Real-time**:
  - Все заказы находятся в общем списке.
  - Изменения, сделанные одним пользователем (например, создание заказа администратором или смена статуса монтажником), **мгновенно** отображаются у всех остальных подключенных пользователей без необходимости обновлять страницу.
  - Архитектура построена на **Supabase Realtime Subscriptions**, что гарантирует актуальность данных.

- **Надежная обработка данных и UX**:
  - Отображение скелетонов во время загрузки данных для предотвращения "пустого экрана".
  - Четкие сообщения об ошибках (отсутствие интернета, ошибки базы данных, неверный формат файла).
  - Автоматическая перезагрузка данных при восстановлении интернет-соединения.
  - Валидация файлов (размер до 5MB, тип) на стороне клиента перед отправкой на сервер.

## Технологии
- **Frontend**: Next.js 15 (App Router), React, TypeScript
- **UI**: Tailwind CSS, ShadCN UI, Lucide Icons
- **Backend & DB**: Supabase (PostgreSQL, Realtime Database, Storage, Auth)

## Развертывание и запуск

1. Подключите репозиторий на GitHub к [Vercel](https://vercel.com/).
2. Vercel автоматически создаст Production-окружение и будет выполнять деплой при каждом `push` в основную ветку.
3. Добавьте переменные окружения в настройках проекта Vercel:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. Приложение будет опубликовано на бесплатном домене вида `*.vercel.app`.
